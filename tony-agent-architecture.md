# Tony's Ruthless Competition Winner Team Architecture

## Overview
This document outlines the architecture for a 5-agent skill system designed to take a startup idea and output a comprehensive, non-generic, high-leverage go-to-market strategy. The system strictly adheres to Tony's Core OS directives: execution over framework theater, no generic filler, verification over confidence, and autonomous testing.

## Agent 1: Competition Crusher
**Goal:** Find the top competitors in 2026 and identify 3 high-leverage product improvements to make the user's solution 10x better.
**Guardrails:**
- Must use live search to find actual competitors, not hallucinated ones.
- Improvements must be specific, actionable, and based on competitor weaknesses (e.g., missing features, poor UX, high pricing).
- No generic advice like "improve customer service."

## Agent 2: ICP Hunter
**Goal:** Identify the top 3 high-leverage, less-saturated Ideal Customer Profiles (ICPs) seeking the solution.
**Guardrails:**
- Must use public data hacks (e.g., analyzing job postings, forum complaints) to find ICPs.
- ICPs must be specific niches, not broad categories (e.g., "B2B SaaS companies").
- Must provide evidence of demand for each ICP.

## Agent 3: Unfair Advantage Marketing Strategist
**Goal:** Provide the top 3 marketing strategies and channels for an unfair advantage.
**Guardrails:**
- Strategies must be unconventional and high-leverage (e.g., scraping specific job boards, intercepting competitor complaints on Twitter).
- No generic advice like "run Facebook ads" or "start a blog."
- Must include actionable steps for execution.

## Agent 4: Script Forge
**Goal:** Generate 6 vertical B2B cold call scripts that pass a strict emotional intelligence audit.
**Guardrails:**
- Scripts must use advanced emotional marketing intelligence and focus on selling the outcome.
- Must include a self-audit for each script, proving it avoids generic filler and uses top B2B cold call strategies.
- Scripts must be tailored to the specific ICPs identified by Agent 2.

## Agent 5: Master Orchestrator
**Goal:** Route the startup idea through Agents 1-4 and compile the results into an ADHD-friendly reference document.
**Guardrails:**
- Must enforce the sequential execution of Agents 1-4.
- Output document must be highly structured, concise, and easy to read (e.g., using bullet points, bold text, and clear headings).
- Must include the strict binary audit from Tony's Core OS.
